# 项目方案文档：多媒体播放器 (MediaPlayer)

## 1. 项目背景

随着数字化娱乐内容的快速增长，用户对本地媒体播放的需求依然旺盛。无论是在线下载的音频文件还是本地视频资源，都需要一个功能齐全、操作便捷的播放器。本项目的目标是开发一款基于C# WinForms的桌面多媒体播放器，支持主流音视频格式的播放与管理。

本项目为《Windows程序设计》课程期末项目，旨在综合运用课程所学的WinForms控件编程、多媒体处理、文件管理、数据库操作等技术，同时实践AI辅助编程（Claude Code）和RDD/TDD开发模式。

## 2. 功能需求分析

### 2.1 核心功能

| 功能模块 | 功能描述 | 优先级 |
|---------|---------|-------|
| 媒体播放 | 支持MP3/WAV/MP4/AVI/MKV等主流音视频格式的播放、暂停、停止 | P0 |
| 播放列表管理 | 添加、删除、清空媒体文件，支持拖放操作 | P0 |
| 播放控制 | 上一曲/下一曲切换、循环播放模式切换 | P0 |
| 音量控制 | 0-100%音量调节 | P1 |
| 进度控制 | 播放进度条显示，支持拖拽定位 | P1 |
| 数据持久化 | 使用SQLite保存播放列表，程序重启后自动恢复 | P1 |
| 批量导入 | 支持打开文件夹批量导入媒体文件 | P2 |

### 2.2 非功能需求
- 界面简洁美观，操作流畅
- 支持高DPI显示
- 内存占用合理，长时间运行不泄漏
- 错误处理完善，异常情况下不崩溃

## 3. 技术选型理由

| 技术 | 选择理由 |
|------|---------|
| **.NET 10 + WinForms** | 使用最新.NET版本，WinForms是Windows桌面应用开发的成熟框架，与课程教学内容完全匹配 |
| **C#** | 课程核心语言，类型安全，语法简洁，支持LINQ和异步编程 |
| **LibVLCSharp** | 基于VLC的.NET绑定，支持几乎所有音视频格式，无需安装额外解码器，API设计良好 |
| **SQLite** | 轻量级嵌入式数据库，零配置，适合本地数据持久化，通过System.Data.SQLite访问 |
| **ADO.NET** | 使用参数化查询进行数据库操作，防止SQL注入，代码可控性强 |

## 4. 系统架构设计

### 4.1 分层架构

```
┌─────────────────────────────────────┐
│          表示层 (UI Layer)           │
│  MainForm.cs  - 主窗体UI与交互逻辑   │
│  控件: VideoView, ListView,          │
│        TrackBar, Button, MenuStrip   │
├─────────────────────────────────────┤
│         业务逻辑层 (BLL)              │
│  PlaylistService.cs  - 播放列表管理  │
├─────────────────────────────────────┤
│         数据访问层 (DAL)              │
│  DatabaseService.cs  - SQLite操作   │
├─────────────────────────────────────┤
│         数据模型层 (Model)            │
│  MediaItem.cs  - 媒体文件实体模型    │
└─────────────────────────────────────┘

外部依赖:
  LibVLCSharp  - 音视频解码与播放引擎
  System.Data.SQLite - SQLite数据提供程序
```

### 4.2 核心类设计

- **Program.cs**: 应用程序入口，配置DPI和视觉样式
- **MainForm**: 主窗体，管理所有UI组件和用户交互
- **Models/MediaItem.cs**: 媒体文件数据模型，包含文件路径、标题、时长等属性
- **Services/PlaylistService.cs**: 播放列表业务逻辑，管理增删改查和播放状态
- **Services/DatabaseService.cs**: 数据库层，封装SQLite的CRUD操作
